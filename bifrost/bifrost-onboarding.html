Domain owner path
1

Understand what gets published

You add a bifrost-spec.yaml to your repo root. It declares your team's domain model: entities, services, commands, queries, events, and relationships. This feeds the BiFrost Catalog, MCP server, and Logos REST API.

Why this matters: AI is now the primary consumer of our information ecosystem. Every unpublished domain object is a blind spot for AI agents, the Nexus Workflow Designer, and cross-team integration.
📄 Read the RFC 📚 Full documentation
2

Generate and validate your spec

Run specgen against your codebase to auto-generate a draft spec, then run bifrost-validator to catch errors before committing. Both tools are available on GitHub Releases.

Terminal
# Auto-generate a spec from your codebase specgen generate --repo-path /path/to/your/repo # Validate before committing ./bifrost-validator validate ./bifrost-spec.yaml --verbose
Review the output. specgen gets you ~80% there. Check descriptions, API mappings, and relationships before committing.
📚 Detailed generation steps ✅ Naming rules and checklist
3

Commit and set up CI sync

Commit your bifrost-spec.yaml and push. Then post your repo URL in #approval-workflow-onboarding to get the GitHub Action added. After that, every merge to main auto-validates and syncs to the catalog.

Git
git add bifrost-spec.yaml git commit -m "feat: add BiFrost domain object spec" git push
4

Check the scorecard

Once your spec is merged, verify your team's completeness on the scorecard. Use the reference checklist to close any remaining gaps.

End user path
1

Get an API key

Request a key in #approval-workflow-onboarding on Slack. You'll get an ipfx_sk_* key to authenticate with the MCP server.

2

Configure your AI tool

Add BiFrost to your IDE's MCP configuration. Pick your tool below.

.cursor/mcp.json
{ "mcpServers": { "ipfx-catalog": { "type": "sse", "url": "https://catalog.bifrost.ogintegration.us/sse?api_key=ipfx_sk_YOUR_KEY" } } }
~/Library/Application Support/Claude/claude_desktop_config.json
{ "mcpServers": { "ipfx-catalog": { "command": "npx", "args": [ "-y", "mcp-remote", "https://catalog.bifrost.ogintegration.us/sse", "--header", "X-API-Key:ipfx_sk_YOUR_KEY" ] } } }
.mcp.json or ~/.claude/settings.json
{ "mcpServers": { "ipfx-catalog": { "url": "https://catalog.bifrost.ogintegration.us/sse", "headers": { "X-API-Key": "ipfx_sk_YOUR_KEY" } } } }
3

Try it out

Ask your AI assistant something like "List all domains in BiFrost" or "What events does AccountsPayable produce?" The MCP server exposes 16 tools for discovery, deep dives, and integration planning.

📚 Full tool reference and example prompts
4

Use it with everything you build

Before building new services, integrations, or features, check BiFrost first. Discover existing domain objects so you reuse instead of duplicate. Let your AI scaffold from real metadata instead of guessing.

  • Search the catalog before creating new entities or events
  • Trace event flows to understand upstream and downstream dependencies
  • Use published APIs and schemas as your integration contracts
